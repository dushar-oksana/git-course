1.Как отказаться от изменений в файле (файлах) находящийся 
в рабочем каталоге (work directory)?
git checkout file_name - эта команда может быть использована если изменения не 
были переданы в статус staged

2.Как вернуть файл из индекса в рабочий каталог?
git restore --staged file_name

3.Как отменить ошибочный commit ?
git reset HEAD^ или git reset HEAD~1

4.Как создать новую ветку и переключится в нее?
git checkout -b branch_name

5.В чем отличение между fast-forward merge  and non fast-forfard merge?
fast-forward merge - когда текузая ветка не имеет допольнительный коммитов по 
сравнениюс веткой которую мы мержим, в данном случае не создается новый коммит, 
а коммиты объединяются в текущей ветке
non fast-forfard merge - если в текузей ветке есть изменения, которых нет в 
ветке, которую мы хотим объединить, выполнится объединение без fast-forfard 
merge. git создаст новый коммит мержа в активную ветку 

6.Когда может возникнуть конфликт слияния веток, как  его разрешить?
если две ветки, которые мы хотим смержить, имеют изменения в одной строке в 
одном и том же файле, или если одла ветка удалила файл, оторый другая ветка 
изменила и тп, может возникнуть конфликт и git спросит какой из двух вариантов 
нужно сохранить. можно использовать команду git rebase branch_name  - git 
скопирует коммиты из текущей ветки и поместит поверз указанной ветки эти коммиты

7.Можно ли удалить ветку? Как? 
в локальном репозитории это можно сделать с помощью команды git branch -d 
branch_name, необходимо сначала выйти из этой ветки с помощью команды git 
checkout other_branch_name
в удаленном репозитории это можно сделать командой git pushorogin -d 
remote_branch_name
